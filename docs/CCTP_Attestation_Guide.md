# CCTP Attestation Guide

## Introduction

Circle's Cross-Chain Transfer Protocol (CCTP) enables the transfer of USDC between different blockchains. A critical
part of this process is the attestation mechanism, which ensures the validity and security of cross-chain transfers.
This guide explains how attestations work in CCTP and how to implement them in your application.

## What is a CCTP Attestation?

When a user burns USDC on the source chain (e.g., Ethereum Sepolia), Circle's attestation service generates a
cryptographic proof that the burn occurred. This attestation is required to mint the equivalent amount of USDC on the
destination chain (e.g., Base Sepolia).

### Key Components

1. **MessageSent Event**: Emitted when USDC is burned on the source chain
2. **Attestation**: Cryptographic proof generated by Circle's attestation service
3. **MessageTransmitter Contract**: Receives the attestation and mints USDC on the destination chain

## Attestation Process

### 1. Burning USDC (Source Chain)

```solidity
// User approves TokenMessenger to spend their USDC
IERC20(usdcToken).approve(tokenMessenger, amount);

// User burns USDC for transfer
ITokenMessenger(tokenMessenger).depositForBurn(
    amount,
    destinationDomain,
    recipient,
    usdcToken
);
```

This emits a `MessageSent` event with:

- `messageBytes`: The encoded message
- `destinationDomain`: The domain of the destination chain
- `recipient`: The recipient address on the destination chain

### 2. Polling for Attestation

After the burn transaction is confirmed, you need to poll Circle's Iris V2 API for the attestation:

```typescript
async function getAttestation(messageHash: string): Promise<Attestation> {
  const response = await fetch(`https://iris-api-sandbox.circle.com/v1/attestations/${messageHash}`);

  if (!response.ok) {
    throw new Error(`Failed to get attestation: ${response.statusText}`);
  }

  const data = await response.json();

  if (data.status !== "complete") {
    throw new Error("Attestation not ready yet");
  }

  return data.attestation;
}
```

Before using the attestation, verify its validity:

### 4. Relaying to Destination Chain

Once you have a valid attestation, call the `receiveMessage` function on the destination chain's MessageTransmitter:

```solidity

    attestation,
    signature

```

This will mint the USDC on the destination chain and transfer it to the recipient.

## Implementation Example

Here's a complete example of how to implement CCTP attestation polling and relaying:

```typescript
import { ethers } from "ethers";
import { CHAINS } from "../config/chains";

interface Attestation {
  messageHash: string;
  status: string;
  attestation: string;
  signature: string;
}

class CCTPMonitor {
  private provider: ethers.JsonRpcProvider;
  private sourceChain: typeof CHAINS.WORLD_CHAIN;
  private destinationChain: typeof CHAINS.BASE_SEPOLIA;

  constructor() {
    this.sourceChain = CHAINS.WORLD_CHAIN;
    this.destinationChain = CHAINS.BASE_SEPOLIA;
    this.provider = new ethers.JsonRpcProvider(this.sourceChain.rpcUrl);
  }

  async monitorAndRelay(txHash: string): Promise<void> {
    try {
      // 1. Get transaction receipt
      const receipt = await this.provider.getTransactionReceipt(txHash);
      if (!receipt) {
        throw new Error("Transaction not found");
      }

      // 2. Extract MessageSent event
      const messageSentEvent = receipt.logs.find((log) => log.topics[0] === ethers.id("MessageSent(bytes)"));

      if (!messageSentEvent) {
        throw new Error("MessageSent event not found");
      }

      // 3. Get message hash
      const messageHash = ethers.keccak256(messageSentEvent.data);

      // 4. Poll for attestation
      const attestation = await this.pollForAttestation(messageHash);

      // 5. Relay to destination chain
      await this.relayToDestination(attestation);
    } catch (error) {
      console.error("Error in monitorAndRelay:", error);
      throw error;
    }
  }

  private async pollForAttestation(messageHash: string): Promise<Attestation> {
    const maxAttempts = 60; // 30 minutes with 30-second intervals
    const interval = 30000; // 30 seconds

    for (let attempt = 1; attempt <= maxAttempts; attempt++) {
      try {
        const response = await fetch(`https://iris-api-sandbox.circle.com/v1/attestations/${messageHash}`);

        if (!response.ok) {
          throw new Error(`HTTP error! status: ${response.status}`);
        }

        const data = await response.json();

        if (data.status === "complete") {
          return data;
        }

        console.log(`Attestation not ready (attempt ${attempt}/${maxAttempts})`);

        if (attempt < maxAttempts) {
          await new Promise((resolve) => setTimeout(resolve, interval));
        }
      } catch (error) {
        console.error(`Error polling for attestation (attempt ${attempt}/${maxAttempts}):`, error);

        if (attempt < maxAttempts) {
          await new Promise((resolve) => setTimeout(resolve, interval));
        }
      }
    }

    throw new Error("Attestation polling timed out");
  }

  private async relayToDestination(attestation: Attestation): Promise<void> {
    const destinationProvider = new ethers.JsonRpcProvider(this.destinationChain.rpcUrl);
    const wallet = new ethers.Wallet(process.env.PRIVATE_KEY!, destinationProvider);

    const messageTransmitter = new ethers.Contract(
      this.destinationChain.messageTransmitter,
      ["function receiveMessage(bytes calldata message, bytes calldata signature)"],
      wallet,
    );

    const tx = await messageTransmitter.receiveMessage(attestation.attestation, attestation.signature);

    await tx.wait();

    console.log("USDC successfully minted on destination chain");
  }
}
```

## Error Handling

### Common Issues

1. **Attestation Not Ready**
   - Cause: Circle's attestation service hasn't processed the burn transaction yet
   - Solution: Implement polling with exponential backoff
   - Typical wait time: 5-40 minutes on testnet

2. **Invalid Attestation**
   - Cause: The attestation doesn't match the message hash or is improperly signed
   - Solution: Verify the attestation before using it

3. **Network Errors**
   - Cause: Issues with Circle's API or network connectivity
   - Solution: Implement retry logic with proper error handling

### Retry Strategy

```typescript
async function getAttestationWithRetry(messageHash: string): Promise<Attestation> {
  const maxRetries = 5;
  const baseDelay = 1000; // 1 second

  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      return await getAttestation(messageHash);
    } catch (error) {
      if (attempt === maxRetries) {
        throw error;
      }

      const delay = baseDelay * Math.pow(2, attempt - 1);
      console.log(`Retrying in ${delay}ms...`);
      await new Promise((resolve) => setTimeout(resolve, delay));
    }
  }

  throw new Error("Max retries exceeded");
}
```

## Best Practices

1. **Use Webhooks**: Instead of polling, consider using Circle's webhook functionality for real-time attestation
   notifications.

2. **Implement Timeouts**: Set reasonable timeouts for API calls to prevent hanging.

3. **Log Everything**: Log all attestation-related events for debugging and monitoring.

4. **Handle Edge Cases**: Consider what happens if:
   - The attestation never arrives
   - The attestation is invalid
   - The destination chain transaction fails

5. **Monitor Performance**: Track attestation times and success rates to identify issues early.
